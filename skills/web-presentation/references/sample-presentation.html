HTML + CSS + JavaScript만으로 구현

오프닝 슬라이드입니다.

왜 바닐라인가?

  • 의존성 0 — 빌드 도구, 번들러, CDN 불필요
  • 단일 HTML 파일로 즉시 실행 가능
  • 라이브러리 API 변경에 영향받지 않음
  • 브라우저 네이티브 기능만으로 충분히 구현 가능
  • 완전한 코드 소유권과 커스터마이징 자유
바닐라 JS를 선택한 이유를 설명합니다.

핵심 기능

  • 키보드 / 터치 / 클릭 네비게이션
  • GPU 가속 슬라이드 전환 애니메이션
  • 진입 시 자동 순차 등장 효과
  • 풀스크린 뷰포트 + 반응형 타이포그래피
  • URL hash 동기화 + 상단 프로그레스 바
주요 기능을 소개합니다.

GPU 가속 전환

// Compositor 스레드에서 처리 — Layout/Paint 없음
.slide {
  transform: translate3d(100%, 0, 0); /* GPU 레이어 활성화 */
  opacity: 0;
  transition: transform 0.6s cubic-bezier(0.25, 0.46, 0.45, 0.94),
              opacity 0.6s ease;
}
.slide.active { transform: translate3d(0, 0, 0); opacity: 1; }
translate3d와 opacity만 애니메이션하여 GPU 가속을 활용합니다.

슬라이드 전환 원리

잠금 메커니즘
  • isAnimating 플래그로 중복 입력 차단
  • transitionend 이벤트로 완료 감지
  • 700ms 안전 타이머 (누락 대비)
성능 최적화
  • will-change 직전 적용·즉시 해제
  • Forced reflow로 애니메이션 보장
  • 인라인 스타일 cleanup 처리
전환 메커니즘의 핵심 개념들입니다.

코드 예시

// transitionend 완료 감지 + 안전장치
const onEnd = (e) => {
  if (e.propertyName !== 'transform') return;
  clearTimeout(safetyTimer);
  cleanup(prevSlide);
};
to.addEventListener('transitionend', onEnd, { once: true });
const safetyTimer = setTimeout(cleanup, 700);
transitionend 이벤트 처리 코드입니다.
